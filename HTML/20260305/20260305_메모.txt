http 프로토콜

- 문자기반 프로토콜
- 파일에 저장된 내용물을 텍스트 기반으로 리턴
- 파일에 저장된 무언가가 실재하는지는 알바 ㄴ
- 포트번호 고정 - 디폴트로 80 사용 - 80 아니면 명시해줘야

상대한테 뭔가 보내려면 상대의 주소, 포트번호 알아야 메시지 보낼수 ㅇㅇ
서로 포트번호에 대해 약속을 해놔야

클라가 서버에 요청 - request
서버가 클라에 응답 - response
요청하기 전에 connection맺고 응답 후에는 끝
서버는 예전의 통신내용 기억 x- 항상 새로운 통신이라 생각 - stateless
프로토콜의 절차 
1. connect
- ip주소와 port번호 제시해야

2. request
- 클라가 서버에 메시지 보냄
3. response
- 응답 보냄
4. 끝

단점 - 곧 또 통신할건데 계속 끊고 다시하면 overload 생김
---> keep alive -- 차피 곧 또하니까 논리적으론 끊겼지만 물리적으론 살아있는 -- 서버도 끊은걸로 ㅇㅇ 예전 정보 잊음
